# Power Outage Monitor

Automated monitoring and calendar integration system for power outage schedules from the Lviv energy company website.  
Fetches, parses, and stores outage schedules, and generates Google Calendar-compatible ICS files.  
Supports group filtering via command-line or JSON file.

---

## Features

- Extracts power outage schedules from [https://poweron.loe.lviv.ua/](https://poweron.loe.lviv.ua/)
- Handles JavaScript-rendered content using Selenium WebDriver
- Parses Ukrainian text and normalizes dates/times
- Stores all data in a local SQLite database
- Generates individual and combined ICS files for Google Calendar
- Supports all-day and timed events
- Maintains event state and handles event deletions
- Group filtering via CLI or JSON file
- Continuous monitoring mode with configurable interval
- Comprehensive logging and error handling
- Exports data to CSV

---

## Requirements

- Python 3.7+
- Google Chrome browser
- ChromeDriver (matching your Chrome version)
- Python packages:
  - selenium
  - pytz

Install dependencies:
bash
```
pip install selenium pytz
```
Installation

Clone the repository:
```
git clone https://github.com/SerhiiYAi/power-outage-monitor.git
cd power-outage-monitor
```
Download ChromeDriver:
https://chromedriver.chromium.org/downloads
Place the executable in your PATH or the project directory.

Usage
Basic Run
bash
```
python power_monitor.py
```
Filter by Groups (CLI)
bash
```
python power_monitor.py --groups 1.1,2.1
```
Filter by Groups (JSON file)
Create a file (e.g., groups.json):

json
{
  "group": ["1.1", "2.1"]
}
Then run:

bash
```
python power_monitor.py --groups-file groups.json
```
Continuous Monitoring
python
```
# In Python interactive or script:
monitor.run_continuous_monitoring(15)  # Every 15 minutes
```
Output

JSON data: json_data/YYYYMMDD_HHMMSS_power_outages.json
ICS files: calendar_events/
Individual event files
Combined all-events file
Cancellation and deletion summary files
Database: power_outages.db
Logs: power_monitor.log
CSV export: power_outages_export.csv
Logging & Filtering
On startup, the script logs whether a group filter is applied:
Groups filter applied: ['1.1', '2.1']
or Executed without groups filter
All major actions and errors are logged to power_monitor.log.
